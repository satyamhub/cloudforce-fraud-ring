# Round 2 Deck Template (TigerGraph Track)

Replace prompts with your content/screenshots. Keep it concise (1–2 bullets per box).

1. Title
   - Team name, members, emails/phone, track: TigerGraph.

2. Problem
   - One-line problem statement.
   - Who is affected + current pain metric.

3. Solution Snapshot
   - One sentence: what your product does.
   - Screenshot or mock (optional) showing the key action.

4. Architecture (diagram)
   - Data source(s) → TigerGraph (schema + queries) → API/UI/ML.
   - Runtime: Docker local; note any cloud pieces if used.

5. Data Model
   - Vertices + attributes; edges + meaning.
   - Why this graph shape fits the problem.

6. Queries / Logic
   - Main GSQL query(ies) and purpose (e.g., recommendation, path, scoring).
   - Sample result JSON or screenshot.

7. Demo Evidence
   - GraphStudio schema screenshot.
   - Loading job success screenshot.
   - Query run screenshot (curl/REST++).
   - (Optional) link to short demo video.

8. Tech Stack
   - TigerGraph 4.2.2, other services, frontend/backend libs, deployment.

9. Impact & Metrics
   - What you can measure now (latency, coverage, accuracy, etc.).
   - Target metric by finale (April 12, 2026).

10. Roadmap to Finale
   - Next 3 deliverables.
   - Risks + mitigations.

11. Team
   - Roles / who owns what.
   - Contact links (GitHub, LinkedIn, Calendly if relevant).
